⚡ 1-Minute Summary

Every programming task starts with problem analysis, not coding. First, you write an algorithm — an ordered list of steps. You can express this as pseudocode (English-like informal language) or as a flowchart (graphical version). Pseudocode is then refined into a detailed algorithm close to actual code. Flowcharts use specific shapes: ovals for start/stop, parallelograms for input, rectangles for processes, diamonds for decisions, and hybrid shapes for output. The four control structures are Sequence (top to bottom), Decision (if/else branches), Loop (repeat while condition), and Case (multi-branch).

Algorithm & Pseudocode

Understanding the building blocks before we write code

📌 What is an Algorithm?

An ordered sequence of steps that solves a problem
Created during the problem-solving phase
Must be precise, unambiguous, and finite
Blueprint before actual coding begins

📌 What is Pseudocode?

An artificial and informal language
Helps programmers develop algorithms
Very similar to everyday English
Not executable — just for planning

Steps in Problem Solving

Step 1: Produce a general algorithm using pseudocode
Step 2: Refine the algorithm successively
Step 3: Get a step-by-step detailed algorithm close to a computer language
Step 4: Implement in actual programming language

📖 Example 1 — Student Grade Checker

Write an algorithm to determine a student's final grade (average of 4 marks) and indicate PASSED or FAILED.

PSEUDOCODE

Input a set of 4 marks Calculate their average by summing and dividing by 4 if average is below 75 Print "FAILED" else Print "PASSED"

DETAILED ALGORITHM

Step 1: Input M1, M2, M3, M4 Step 2: GRADE ← (M1+M2+M3+M4)/4 Step 3: if (GRADE < 75) then Print "FAILED" else Print "PASSED" endif End of Program.

📖 Example 2 — Even or Odd Checker

BEGIN PRINT "Enter a number: " READ number Get number % 2 to get the remainder IF Remainder is equal to 0 THEN PRINT "EVEN." else PRINT "ODD" ENDIF END
The % (modulo) operator returns the remainder of division. If remainder = 0 → EVEN; else → ODD

📏 Flowchart Rules (Must Know!)

Must have a START and STOP
All steps must connect — no hanging steps
Shows logic solution graphically
Emphasizes individual steps and their interconnections
Control flow goes from one action to the next
Diamond always has exactly 2 exits (Y/N)

Control Structures

The four building blocks of algorithmic logic

1️⃣ Sequence

Statements executed one after another, in order
No branching, no skipping
Flow: top → bottom, left → right
Statement 1 ↓ Statement 2 ↓ Statement 3

2️⃣ Decision (Selection)

Branching based on a condition
Uses IF/THEN/ELSE
Diamond symbol with 2 exits: YES and NO
IF (condition) THEN then-statement(s) ELSE else-statement(s) ENDIF

3️⃣ Loop (Repetition)

Repeats statements while a condition holds
While Loop: Check condition FIRST, then execute
Do-While Loop: Execute FIRST, then check
// While Loop WHILE (condition) DO loop-statement(s) ENDWHILE // Do-While Loop DO loop-statement(s) WHILE (condition)

4️⃣ Case (Multi-Selection)

Like multiple if-else branches
An expression is evaluated
Each possible value has its own branch
"otherwise" = default branch
CASE expression OF choice_a: process_a choice_b: process_b otherwise: default_process ENDCASE

⚖️ Comparison: While vs Do-While

While Loop

Condition checked BEFORE executing
Body may never execute if condition is false from start
Use when: You might need 0 iterations

Do-While Loop

Task executed FIRST, condition checked after
Body always executes at least once
Use when: You need at least 1 iteration

Problem A — Quadratic Roots

For ax² + bx + c = 0, the discriminant D = b² - 4ac determines root nature.

ALGORITHM

Step 1: Input a, b, c Step 2: D ← b² - 4*a*c Step 3: if D > 0 then x1 ← (-b + √D) / (2a) x2 ← (-b - √D) / (2a) Print "Real & Unequal Roots" Print x1, x2 else if D == 0 then x ← -b / (2a) Print "Real & Equal Roots" Print x else Print "Imaginary Roots" endif

ROOT NATURE GUIDE

D > 0 → Two distinct real roots (real & unequal)
D = 0 → One repeated real root (real & equal)
D < 0 → No real roots (imaginary / complex)

Flowchart hint: Use NESTED decisions!

Decision 1: Is D > 0?
→ YES: Real & Unequal
→ NO: Decision 2: Is D = 0?
  → YES: Real & Equal
  → NO: Imaginary
